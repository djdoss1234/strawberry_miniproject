# 단일 딸기에서 시작한 설명 가능한 수확 모션

## WHY

저 민석은 화면에서 딸기를 검출하는 데서 끝나는 데모가 아니라, 실제 줄기를
잡아 과실을 분리하는 시스템을 만들고자 했습니다. 가장 쉬운 SW 단일 과실에서도
로봇은 IK branch를 크게 바꾸거나 잎을 밀었고, `pick_complete`가 출력돼도 실제
수확은 실패했습니다. 이 차이를 해결하지 않으면 군집·가림 환경으로 확장할 수
없다고 판단했습니다.

## HOW

실패를 perception, planning, execution, verification 네 단계로 나누고 매 실행을
JSONL과 Git commit으로 연결했습니다. cuRobo는 긴 구간의 IK·충돌·관절 branch
검증에 적합하지만 최종 TCP 직선성을 보장하지 않는다는 점을 확인했습니다.
따라서 pre-approach까지는 cuRobo와 MoveSplineJoint를 사용하고, 줄기 근처에서는
정지 후 Doosan MoveLine으로 직선 진입하도록 역할을 분리했습니다.

실제 로봇 관찰을 바탕으로 목표 높이를 `+30mm`, 추가 진입을 `65mm`, 분리
동작을 BASE `-Z 40mm`로 조정했습니다. 1-step 접근으로 시간을 줄이려다 정확도가
떨어진 실패도 확인했고, 최종 접촉 정확도를 우선해 2-step 구조를 복원했습니다.
Claude Code와 Codex는 로그 비교와 가설 탐색에 활용했으며, 실기 적용과 최종
파라미터 결정은 제가 직접 내렸습니다.

## WHAT

SW 단일 과실에서 줄기 파지와 분리의 육안 성공 사례를 확보했고, 최신 완료
시퀀스는 약 `36.4초`가 소요됐습니다. 접근·파지·분리·scan pose 복귀 이벤트를
재생 가능한 JSONL로 남겨 실패 원인을 추적할 수 있게 했습니다.

자동 그리퍼 판정은 아직 `GRASP_UNVERIFIED`이며 반복 성공률, 3D target 오차,
손상률은 **측정 필요**입니다. 다음 단계는 cycle time 단축과 30회 반복 정량
검증, marker 기반 place, NE 군집 및 NW 가림 셀 수확입니다.

## 핵심 수치

| 지표 | 현재 값 |
| --- | --- |
| 최신 완료 시퀀스 | 약 `36.4초` |
| grasp Z bias | `+30mm` |
| 추가 진입 | `65mm` |
| 분리 동작 | BASE `-Z 40mm` |
| 육안 결과 | SW 단일 과실 파지·분리 성공 사례 확보 |
| 자동 성공률 / 오차 / 손상률 | 측정 필요 |
